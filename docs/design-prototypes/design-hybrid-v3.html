volatile 能保证线程安全吗？为什么？ ⏱ 2min
参考要点
答题思路
可见性 ✓ 有序性 ✓ 原子性 ✗ → 不能单独保证线程安全。
需配合 synchronized 或 Atomic 类保证原子性。
解释指令重排及 volatile 的禁止重排原理 ⏱ 3min
参考要点
答题思路
编译器/CPU 可能重排指令优化性能。volatile 通过插入内存屏障（StoreStore + StoreLoad）禁止重排。
双重检查锁单例为什么需要 volatile？ ⏱ 2min
参考要点（已展开）
答题思路
new 对象非原子操作（分配内存→初始化→赋值），可能重排为"分配→赋值→初始化"，此时其他线程拿到未初始化对象。volatile 禁止重排。
